# Red Team Review & Adversarial Attack Resilience Report

This report assesses the ranker's resilience against common search cheats, fake AI resumes, and timeline honeypots.

## Adversarial Defenses Audit Table

| Threat Vector | Attack Scenario | Mitigation Strategy | Defense Status |
| :--- | :--- | :--- | :--- |
| **Keyword Stuffing** | Candidate list 100 AI/ML terms in profile summary without career depth | Cross-Encoder context matching + Role-Specific experience depth score gating | ✅ SECURE |
| **Title Traps** | Marketing Manager with a headline like 'AI Researcher' | Strict 5-tier role taxonomy gate using current_title parsing | ✅ SECURE |
| **Honeypot Timelines** | Synthesized career history showing overlapping or impossible dates | Overlap & chronological timeline check returning continuous risk score | ✅ SECURE |
| **Trust Spammers** | Low-relevance candidate with verified contact channels | Trust score acts as a light additive separator, not a multiplier | ✅ SECURE |
| **Location Spoofers** | Abroad candidate claiming relocation capability | Relocation gets a minor boost, but local candidates strictly outrank equal-profile remote ones | ✅ SECURE |

## System Vulnerability Assessment
1. **Semantic Drift Resilience**: By using a hybrid retrieval mechanism (80% Dense Semantic / 20% BM25 Lexical), the system is highly robust to synonym mismatch while retaining keyword precision.
2. **Honeypot Suppression**: The Risk score successfully penalizes candidates who list advanced skills like FAISS or Sentence-Transformers but show zero duration or low skill assessments in those areas.
3. **Ranking stability**: The soft division scale factor (1.30) ensures that high relevance scores do not pool/plateau at 1.0, preserving relative ordering and NDCG optimality.
